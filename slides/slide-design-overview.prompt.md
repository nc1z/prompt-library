Create a visual design overview for a presentation using image generation.

Start from an approved or draft slide overview. If no overview is available, ask the user to provide one or run the slide overview prompt first.

Before generating anything, confirm the available inputs:

- Slide overview or outline.
- Theme, audience, tone, and any brand constraints.
- Preferred format or aspect ratio, if known.

If critical inputs are missing, ask for them first. Otherwise proceed.

Design workflow:

1. Generate a small set of image-based design directions.
   - Use image generation for visual mockups or style frames.
   - Show enough variation for the user to compare direction, layout, typography, color, and visual language.
   - Keep the designs aligned with the requested theme and presentation topic.

2. Present a concise design overview.
   - Summarize each design direction in 2-3 bullets.
   - Explain what kind of presentation each direction suits best.
   - Do not generate the full deck yet.

3. Ask for user review.
   - Ask whether the user wants changes or approves one direction.
   - If changes are requested, revise the design overview and mockups before continuing.

4. After approval, offer these output options:
   - `1. Generate individual images in PNG`
   - `2. Generate in Google Slides`
   - `3. Generate in PPT`
   - `4. Generate in PDF`

Rules:

- Do not create the final deck before the user approves a design direction and chooses an output option.
- Keep the design overview short and decision-oriented.
- Make the mockups presentation-specific, not generic decorative art.
- Ensure visual concepts can realistically be implemented across the full slide outline.
- Preserve any user-provided brand, audience, format, or accessibility constraints.
