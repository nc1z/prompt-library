 Read the PRD/spec in this repo and create a PLAN.md for a demo-able enterprise POC.

  Structure it as incremental, commit-friendly sections with checkbox subtasks. Each section should have:
  - A clear feature/theme heading.
  - A one-sentence goal.
  - Concrete `- [ ]` tasks.
  - A suggested commit message.

  Make sure the plan covers:
  - Repository baseline and docs.
  - Demo fixture data.
  - Core pipeline or app logic.
  - Codex App-style human review outputs.
  - Codex CLI-style repo edits or generated artifacts.
  - Safety and approval boundaries.
  - UI or monitoring/demo experience.
  - Tests, evals, or acceptance checks.
  - Reset/replay workflow.
  - Enterprise demo narrative.
  - Final acceptance pass.
